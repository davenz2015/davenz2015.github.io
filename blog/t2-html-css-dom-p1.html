Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

I would maybe use the analogy of a house. The html is the basic structure of a webpage and has different parts to it, much the same as framing gives the house its basic structure and is comprised of different patrs too - bedrooms, kitchen, bathrooms etc.

CSS styles a webpage and defines how it looks - colors, shapes, positions of different elements. Not every webpage is decorated the same and neither is every house. So with a house that would be like whether you want paint or wallpaper on the walls, carpet or tiles on the floor and what colors you would want to use.

The DOM (Document Object Model) is an program that shows the structure of a webpage and can access and manipulate it. This would be like the house plans on the Architects computer. It shows all of the house, and they can make changes to it and immediately see what it will look like with those changes incorporated.

What is meant by boxifying design?

If you get a web design brief the first thing you do is boxify it. A webpage is made up of boxes and thinking in terms of boxes makes life alot easier. When we boxify a design we draw a box around every visual element, starting with the biggest box you can see and working toward smaller boxes.

What is the box model?

As above, everything on a webpage is made up of boxes. The box model is a box that wraps around every html element, allows us to define space between elements and comprises of four things;

  1. Margins - an empty area seperating the element from its neighbours.
  2. Borders - a border that goes around padding and content.
  3. Padding - the area between the content and border.
  4. Content - contains the actual content, text and images.
